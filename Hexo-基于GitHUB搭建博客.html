1. 安装Hexo

在开始之前你需确认已经安装了node.js环境，这就好比你要装QQ必须先装Windows一样，可以简单的理解为Hexo是基于node.js

下载Hexo到本地

在实际开始之前，首先要把Hexo文件下载到本地，或者叫做拉取到本地，这就好比你Windows装好了，要装QQ，你还是要去下载QQ的安装程序到电脑上一样

步骤如下

  1. 新建一个目录用于存放Hexo的文件
  2. 然后再地址栏输入CMD，以便直接定位到该目录，当然你硬是敲命令我也没意见

安装Hexo模块

现在开始正式的安装了，我们将把下载的代码解压缩，和云端联动，初始化的过程有两种，一种是通过Git，一种是通过Node.JS，两种办法目前看来没有区别

无论是cmd运行Node.JS，还是进入hexo文件夹内右键打开Git Bash Here，安装模块的命令相同

  1. 安装模块的命令为npm install -g hexo-cli
    1. 这是原版命令，如果安装缓慢可以更换国内源
    2. 国内源其实是安装了cnpm工具，之后通过cnpm进行安装
    3. 如果安装了cnpm，之后的命令都会令开头都会多个n
    4. 命令会变更成cnpm install -g hexo-cli
    5. 本例中并没有咏唱，直接完成了下载操作，怎么可能呢？！
  2. 安装完成后，样张如下
  3. 然后可以通过hexo -v检查一下模块是否安装成功，如果成功会出现一堆模块的版本信息

初始化Hexo目录

  • 在node.js下，安装了模块并不等于就有了hexo，就好比我下载好了QQ，只要我不双击装，那还是等于没有
  • 初始化这个过程可以在Git下面进行，也你能在CMD定位到目录进行，下面以CMD为例子

和Git初始化本地库一样，都是在本地仓库目录中使用init命令，只是配合Hexo后，命令有所不同

初始化成功后，hexo文件夹内会出现如下的文件
node_modules 依赖包
public存放生成的页面
scaffolds生成文章的一些模板
source用来存放你的文章
themes放下下载的主题
_config.yml博客的核心配置文件（设置主体、标题等属性）

启动Hexo主程序

听起来是不是很牛逼，不，别激动，这其实只是在本地测试一下，稍微感受一下，之后有问题也可以先本地测试，也算是多了一个故障排除的参照

初始化好之后可以什么都不做，直接运行服务，然后查看网站页面，我想这是你等了很久的

  1. 在CMD中运行hexo server，注意需要在安装了Hexo的文件夹进行，本例是在C:\Hexo
  2. 根据提示访问网址，他会给出一个内网地址，一般是http://localhost:4000
  3. 如果端口4000和本地已有端口冲突，那么可以在启动的时候加上端口，比如hexo server -p 5000
  4. 查看网站的时候CMD窗口不能关闭，或者按Ctrl+C终止
  5. 激动吗？还没完呢

正式部署Hexo

听起来是不是超级牛逼与令人兴奋的一个说法？是的，这很让我激动，其实之前都只是在本地测试，而下面要让他通过GitHubPages，真正的变成一个可以访问的网站

修改本地配置文件

  1. 主要配置网站根目录下的_config.yml 配置文件，最好别用记事本，用VSCode或者其他文本编辑器都行
  2. 一定要在先安装这个这个git插件，否则会提示找不到工具
    1. npm install hexo-deployer-git –save
  3. 再次确认有没有按照==[[Git-Win环境搭建#设置全局用户信息]]==中的内容
  4. 再次确认，==SSH秘钥==是否设置完成，如果又忘了，点这个[[IT-如何创建SSH秘钥]]看看。。ORZ
  5. 打开文件之后在文件的最下方，也就是104行开始进行更改，更改参考内容参考如下

使用Git的方式

使用http的方式

就和Git命令一样，其实有如上两种上传方式
1. 用git方式，需要用到SSH秘钥，要提前上传，好处是不用每次都输入密码
2. http方式，好处是不用SSH秘钥，问题是每次都要输入密码
我喜欢ssh方式，看起来高端啊~！
